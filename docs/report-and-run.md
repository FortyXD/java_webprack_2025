# Отчет по проекту и запуск сайта

## Общая информация

Проект представляет собой web-приложение для учета банковских отделений, клиентов, счетов и операций по счетам. Работа выполнена Моисейкиным Андреем Денисовичем, группа 328.

Приложение реализовано как Java Web-проект с использованием Spring MVC, Hibernate ORM, JSP-представлений и небольшого React/Tailwind frontend-слоя для сборки статических ассетов. Данные хранятся в PostgreSQL.

## Использованные технологии

- Java 17 — основной язык разработки;
- Spring MVC 6 — обработка web-запросов и маршрутизация страниц;
- Hibernate ORM 6 — работа с БД через сущности и DAO;
- PostgreSQL — реляционная база данных;
- JSP — серверные страницы интерфейса;
- React 19, Vite, Tailwind CSS — сборка frontend-ассетов;
- Ant — основная сборка, запуск сайта и тестов;
- Gradle — альтернативная сборка DAO-части и отчетов;
- TestNG — модульные DAO-тесты;
- Selenium + Pytest — системные UI-тесты.

## Структура проекта

- `src/ru/javaprac/bank/entity` — Hibernate-сущности предметной области;
- `src/ru/javaprac/bank/dao` — DAO-классы для доступа к данным;
- `src/ru/javaprac/bank/web` — web-контроллеры, JSON API, сервис и встроенный Tomcat;
- `webapp/WEB-INF/jsp` — JSP-страницы пользовательского интерфейса;
- `frontend/src` — React/Tailwind frontend-часть, собираемая в `webapp/assets`;
- `sql` — SQL-скрипты создания, очистки, наполнения и просмотра БД;
- `test` — модульные тесты DAO-слоя;
- `system-tests` — Selenium-сценарии для проверки web-интерфейса;
- `docs` — документация, отчетные материалы и описание системных тестов.

## База данных

В базе данных используются следующие основные таблицы:

- `departments` — банковские отделения;
- `clients` — клиенты банка;
- `currencys` — справочник валют;
- `accounts` — банковские счета;
- `credit` — параметры кредитных счетов;
- `deposit` — параметры депозитных счетов;
- `saving_account` — параметры накопительных счетов;
- `account_operations` — история операций по счетам.

Схема поддерживает связи между отделениями, клиентами, счетами и операциями. Для клиентов заданы уникальные ограничения на паспорт и СНИЛС. Для счетов задан уникальный специальный номер и статусы `active`, `closed`, `suspended`.


## Бизнес-правила

- отделение нельзя удалить, если к нему прикреплены клиенты;
- клиента нельзя удалить, если у него есть открытые счета или ненулевой баланс;
- счет нельзя закрыть, если баланс не равен нулю;
- при списании проверяется достаточность средств;
- при создании клиента проверяются обязательные поля и уникальность паспорта/СНИЛС;
- при создании счета учитывается его тип: кредитный, депозитный или накопительный.

## Тестирование

В проекте предусмотрены два уровня тестирования:

1. DAO-тесты на TestNG проверяют работу слоя доступа к данным, сущности и статистические запросы.
2. Системные Selenium-тесты проверяют пользовательские сценарии в браузере: загрузку главной страницы, поиск отделений, создание отделений/клиентов, создание счетов и выполнение операций.

Команды тестирования:

```bash
ant test
ant system-test
```
## Итог

В результате разработано web-приложение для учета банковских данных с полноценной серверной частью, web-интерфейсом, PostgreSQL-схемой, тестовыми данными и автоматизированными тестами. Проект можно запускать локально через встроенный Tomcat командой `ant run-web`, а также собирать в WAR-файл для развертывания во внешнем Tomcat.
